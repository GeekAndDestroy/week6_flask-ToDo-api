Kekambas ToDo API

API Documentation

GET /tasks
  • Authentication: None
  • Example Payload: N/A
  • Output: Get all tasks from the task table in a list of dictionaries.
GET /tasks/complete
  • Authentication: None
  • Example Payload: N/A
  • Output: Get all COMPLETED tasks from the task table in a list of dictionaries.
GET /tasks/incomplete
  • Authentication: None
  • Example Payload: N/A
  • Output: Get all INCOMPLETE tasks from the task table in a list of dictionaries.
GET /tasks/<task_id>
  • Authentication: None
  • Example Payload: N/A
  • Output: Get a task in dictionary form based on the ID.
POST /tasks
  • Authentication: Bearer Token
  • Example Payload: { "title": "Example Title", "description": "Example Description" }
  • Output: Create a new task with a title and description, returns the new task with a 201 status.
PUT /tasks/<task_id>
  • Authentication: Bearer Token
  • Example Payload: { "title": "Example Title", "description": "Example Description" }
  • Output: Update a task by id, task must be created by user trying to update.
DELETE /tasks/<task_id>
  • Authentication: Bearer Token
  • Example Payload: N/A
  • Output: Delete a task by id, task must be created by user trying to delete.
GET /me
  • Authentication: Bearer Token
  • Example Payload: N/A
  • Output: return the authenticated user with only their tasks.
GET /me/complete
  • Authentication: Bearer Token
  • Example Payload: N/A
  • Output: return the authenticated user with only their COMPLETED tasks.
GET /me/incomplete
  • Authentication: Bearer Token
  • Example Payload: N/A
  • Output: return the authenticated user with only their INCOMPLETE tasks.
POST /users
  • Authentication: None
  • Example Payload: { "username": "user123", "email": "user123@example.com", "password": "securepassword123" }
  • Output: Create a new user with a username, email, and password.
GET /users/<task_id>
  • Authentication: None
  • Example Payload: N/A
  • Output: Get a user by id in dictionary form or return a 404 status.
PUT /users
  • Authentication: Bearer Token
  • Example Payload: { "username": "user123", "email": "user123@example.com" }
  • Output: Update a user by id, user must be trying to update itself.
DELETE /users
  • Authentication: Bearer Token
  • Example Payload: N/A
  • Output: Delete a user by id, user must be trying to delete itself.
GET /token
  • Authentication: Basic Auth
  • Example Payload: N/A
  • Output: returns a token based on username and password.
